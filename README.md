# Pixelit

A web app for playing nonograms — those picture-logic puzzles where you fill in squares on a grid using number clues, and a little pixel-art image reveals itself when you get it right.

## What it's about

Pixelit is built around one idea: make nonograms a social, endless experience.

Instead of a fixed book of puzzles, Pixelit generates new boards on demand across a range of difficulties, so there's always something fresh to solve. It also lets players design their own puzzles and share them with the community — so the catalog keeps growing on its own.

Every solve is timed and scored. Each puzzle has its own leaderboard, and a global leaderboard tracks the top solvers overall. Your profile keeps your stats and a history of everything you've finished, so you can see yourself getting faster over time.

## What you can do

- **Play** — pick a puzzle from the catalog, filter by difficulty, and solve at your own pace. Unfinished boards save automatically so you can come back later.
- **Create** — design your own nonogram and publish it for others to try.
- **Compete** — climb the per-puzzle and global leaderboards.
- **Track** — view your solve history, times, and stats on your profile.

## How it's built

Pixelit is a TypeScript monorepo (Nx) with a React + Vite frontend and a NestJS backend. Puzzles are generated by a separate Spring service, stored in Postgres, and cached through Redis. Auth is JWT-based. The UI is built on Tailwind and Radix.

```
apps/
  client/   React frontend
  server/   NestJS API
libs/
  types/    shared types
  ui/   shared UI components
```

## Running it

The fastest way to try it locally:

```bash
docker network create nonogram-network
docker compose up
```

Client at `localhost:4200`, API at `localhost:3000`. For local (non-Docker) dev you'll need Postgres, Redis, and an `.env` in `apps/server/` — see `docker-compose.yml` for the expected variables.
